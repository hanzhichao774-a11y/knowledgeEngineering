流程编排

管理智能体定义任务调用链 · Agent as Tool 模式
💡
编排原则：管理智能体是唯一的调度中心
所有子智能体被包装为"工具"（Agent as Tool），由管理智能体统一调用。子智能体之间不能直接互调， 第三方 API 也只能通过管理智能体的"外部调用工具"来访问。这确保了成本可控、流程可审计、结果可追溯。

🔒
安全策略：外部调用集中管控
所有外部 API 调用必须通过管理智能体的"外部调用工具"进行。子智能体无权直接调用任何第三方服务。 这避免了"随意外挂一个智能体就能调第三方"的成本失控问题。每次外部调用都有审计日志。
